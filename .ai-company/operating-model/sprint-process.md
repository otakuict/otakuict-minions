# Sprint Process

Use this process for AI company work.

## 1. Intake

Capture:

- objective
- context
- constraints
- deadline or timebox
- expected outputs
- definition of done

If details are missing, make conservative assumptions unless the risk is high.

## 2. Role Selection

Choose only the roles needed. Common roles:

- Company Lead: owns plan, coordination, integration, memory.
- Product Manager: scope, users, requirements, acceptance criteria.
- Developer: implementation, technical design, tests.
- Data Engineer: data sources, schemas, pipelines, analytics.
- QA Reviewer: bugs, risk, regressions, test gaps.
- Marketing Strategist: positioning, messaging, campaigns, launch assets.

## 3. Sprint Plan

Create a small plan with:

- sprint goal
- task list
- owners
- outputs
- acceptance criteria
- risks

## 4. Execution

Work in bounded tasks. Each task should produce an artifact, code change, analysis, or decision.

## 5. Review

Review against acceptance criteria. Verify important claims with files, tests, data, screenshots, or sources.

## 6. Memory Update

At the end, update:

- `memory/current-state.md`
- `memory/work-log.md`
- `memory/decisions.md` if a meaningful decision was made
- `memory/open-questions.md` if something remains unresolved
- `memory/lessons-learned.md` if a reusable pattern was discovered
